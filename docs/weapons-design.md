# Идеи по оружию / механикам — рабочий блокнот

> Цель: разнообразить гемплей. Сейчас вся техника на гусеницах (это не меняем),
> и все стреляют либо снарядом, либо ракетой — гемплейно одинаково («отними HP по точке»).
> Хотим механики, которые открывают **новые оси** гемплея и **сочетаются** друг с другом.

## Контекст и ограничения (что определяет дизайн)

Уникальное в движке — оружие должно играть именно с этим, а не с числом урона:

1. **Танки собраны из мелких кусочков** (модульное тело, Rapier-суставы).
2. **Реальная физика Rapier** — импульсы, массы, суставы, трение, стены.
3. **Гекс-сетка** — дискретное пространство, расстояния, соседи, зоны.
4. **Обломки персистентны** — убитые/отстреленные куски **остаются на поле боя**.
   На карте копится «мусорное поле». ← НЕДООЦЕНЁННАЯ ось, развиваем отдельно.

**Главное ограничение — RL.** Модель обучается играть за танки (PPO). Значит каждая механика обязана быть:
- **Выразима как действие** — дискретное/непрерывное действие с понятными параметрами
  (направление прицела, целевой гекс, вкл/выкл). Без скрытой магии.
- **Читаема из состояния** — эффект выражен компонентами, которые попадают в observation
  (`Disabled`, `Tethered`, `Slippery`…), чтобы агент мог на него реагировать.
- **С осмысленным reward-сигналом** — польза действия должна проявляться в наблюдаемом
  изменении мира, иначе агент не научится.
- ⚠️ Механики «скрытия информации» (дым, отключение сенсоров) = **частичная
  наблюдаемость**. Обучаемо, но усложняет PPO (нужна память/маскирование наблюдений).
  Помечаем такие отдельно.

## Реестр механик

Статусы: ✅ принято · 🤔 на рассмотрении · ❌ отклонено · 💡 новое (не обсуждали)

### ✅ Грави-пушка (force, не урон)
Не наносит урон — **толкает/тянет/смещает**. Вмазать врага в стену, спихнуть с укрытия/в зону,
сорвать прицел, столкнуть в обломки. Через Rapier — это импульс по телу/куску.
- **ECS:** система навешивает импульс; опц. компонент `Pushed`/`Tethered` если тянем тросом.
- **Действие (RL):** направление + сила (или «тянуть/толкать»). Понятно.
- **Observation:** агент видит свою скорость/смещение, близость стен.
- **Комбо:** масло (скользко → дальше летит) · обломки (есть во что вмазать) · край зоны.

### 🤔 ЭМИ-пушка (отключение, не урон)
Временно отрубает **модуль** врага. Ключевой вопрос — что именно отрубаем:
- **Передвижение** (гусеницы) — враг застывает на N тиков. Окно для добивания. ← нравится.
- Пушку — не может стрелять.
- Сенсоры — не видит (⚠️ частичная наблюдаемость, см. ограничение RL).
- **ECS:** компонент `Disabled { what, untilTick }`; системы движения/стрельбы пропускают
  сущности с этим компонентом (existence-based, без `if(type)`).
- **Действие (RL):** прицел по врагу. **Observation:** агент видит свой `Disabled`. Очень читаемо.
- **Комбо:** ЭМИ (стоит) → таран/грави (добей, пока не двигается).

### 🤔 Дымовая завеса (отрицание информации)
Заливает несколько гексов дымом. Снаружи не видно, что внутри; изнутри не видно наружу.
Area denial по видимости, а не по урону.
- ⚠️ **Частичная наблюдаемость** — самый дорогой по RL пункт. Для PPO это маскирование
  части observation. Обучаемо, но требует памяти у политики и аккуратного reward.
- **ECS:** дым — сущности/состояние на гексах (`Smoke { untilTick }`); система видимости
  режет лучи/наблюдения через задымлённые гексы.
- **Комбо:** дым + отступление · дым перед рывком · дым, чтобы скрыть мину.

### ✅ Морозящая пушка (множитель скорости)
Снижает **коэффициент скорости движения и скорости поворота пушки** у врага.
Не урон — «загустевание». Меняет **цвет кусочков** (визуальный + читаемый признак).
- **ECS:** компонент `Chilled { moveMul, turnMul, untilTick }`; системы движения/поворота
  башни умножают на коэффициент. Цвет — рендер-система реагирует на наличие `Chilled`.
- **Действие (RL):** прицел по врагу. **Observation:** агент видит свои множители < 1.
  Очень читаемо, плавный сигнал (не бинарный, как ЭМИ).
- **Комбо:** заморозил (медленно) → грави-пушка/таран легко добивает; парно с огнемётом.

### ✅ Огнемёт (DoT — зеркало морозилки)
Обратная история к морозилке: вместо замедления — **урон по времени** (DoT) + цвет кусочков.
Симметричная пара через один механизм (накладываем состояние, система тикает эффект).
- **ECS:** компонент `Burning { dps, untilTick }`; система тикает урон. Цвет — как у `Chilled`.
- **Действие (RL):** конус/направление. **Observation:** свой `Burning`, падающее HP.
- **Комбо:** поджечь обломки-«бочки» (детонация поля) · масло + огонь (лужа загорается).

### ✅ Хиллер-техника (собирает куски и лечит)
Поддержка: ездит по полю, **собирает обломки** (магнит-подбор) и **чинит союзников/себя**
этим «материалом». Завязана на ось поля обломков — обломки становятся ресурсом-ремкомплектом.
- **ECS:** `Repairing { targetEid, rate }` + запас «материала» из подобранных кусков.
  Лечение = добавление/восстановление модулей союзнику.
- **Действие (RL):** подбор (наезд) + выбор цели починки. **Observation:** запас материала,
  HP союзников. Reward явный: подобрал → починил → союзник крепче.
- **Комбо:** хиллер за спиной фронта · враг целит хиллера first (приоритет целей).

### 🤔 Мили-танк (пила/бур, отрыв модулей)
Ближний бой, отрывает конкретные куски (гусеница → нет хода; пушка → нет стрельбы).
Заметка пользователя: *«это хорошо, но по сути не сильно меняет идею»* → низкий приоритет
как отдельный юнит, но «локальный урон по модулю» как **свойство** может пригодиться.

## 💡 Ось «поле обломков» (развиваем — это и есть скрытый козырь)

Раз куски остаются на карте, поле со временем превращается в свалку. Это меняет всё:
обломки = укрытия, патроны, препятствия, ресурс. Механики вокруг этого почти бесплатны
(данные уже есть на карте) и отлично читаются агентом (обломки = сущности в observation).

- **✅ Грави-пушка = волна «всё перед собой швыряет».** УПРОЩЁННЫЙ вариант (так и делаем):
  не «прицелься в кусок и подними», а конус/волна — **всё, что перед пушкой (обломки, враги),
  отлетает по направлению волны и домажит** при ударе. Один action (направление), без выбора
  конкретного объекта → проще для RL. Боезапас — это поле боя, бесплатно.
- **💡 Сгрести обломки в стену (укрытие из мусора).** Грави-пушкой/магнитом стянуть куски
  на соседние гексы → временное укрытие, блокирующее снаряды и линию видимости.
  Терраформинг из хлама. Агент учится строить позицию, а не только стрелять.
- **✅ Магнит-подбор / «каннибализм».** Проехать по обломкам и **прирасти** куском
  (чинишься/растёшь, собирая хлам). Reward очень явный: подобрал → стал крепче.
  Базовый механизм и для хиллер-техники (та же сборка материала, но тратит на союзников).
- **💡 Обломки как естественное препятствие для движения.** Уже следует из физики, но можно
  сделать осознанной механикой: завалить проход обломками, чтобы отрезать врагу путь по гексам.
- **💡 Детонация обломков.** Некоторые куски (с топливом/боезапасом) можно поджечь выстрелом —
  поле боя становится минным само по себе. Грави-пушкой подкатил «бочку» к врагу → подорвал.

## 💡 Другие неисследованные оси (накидываю, для обсуждения)

- **💡 Магнит / поляризация (притяжение↔отталкивание).** Обобщение грави-пушки: режим
  «притянуть» собирает обломки/врагов в кучу, «оттолкнуть» расталкивает. Один action,
  два знака. Хорошо для RL (непрерывный параметр).
- **💡 Масло/скользкий гекс.** Не урон — меняет трение на гексах. Враг не может затормозить/
  развернуться. Чистое комбо-топливо для грави-пушки и тарана.
- **💡 Гарпун/трос (tether).** Joint между своим телом и куском врага: подтянуть к себе,
  перевернуть лёгкого, удержать на месте. `Tethered { otherEid }` — читаемо.
- **💡 Якорь/стойка.** Своё действие: «вкопаться» — резко поднять трение/массу, чтобы
  тебя не сдвинули грави-пушкой/тараном. Контр-механика к force-оружию (камень-ножницы).
- **💡 Замедляющее поле/смола на гексах.** Зона, где скорость падает. Контроль территории
  без урона, читаемо агентом (своя скорость в observation).

## 💡 Раунд 2 — оси, которые мы ещё не трогали

Группирую по тому, какое **новое решение** появляется у агента. Каждая — чистый action +
читаемое состояние.

### Ось «линия видимости» (контр-мета к укрытиям/обломкам)
- **💡 Навесное/мортирное (indirect fire).** Летит по дуге на **целевой гекс**, не нужна
  прямая видимость. Бьёт за стенами/обломками. Появляется решение «прямой выстрел или
  навес», и это **контрит** мету укрытий из мусора. RL: action = целевой гекс (дискретно,
  очень чисто). Минус — отложенный reward (снаряд летит).
- **✅ Рельса/пробивной (piercing).** Сверхскоростной выстрел по **прямой линии**, проходит
  сквозь несколько кусков/врагов. Награда за **выстраивание врагов в линию** — новая
  позиционная задача. Читается как луч-линия.

### Ось «активная защита» (в пику пассивным укрытиям из обломков)
- **💡 Разворачиваемый щит/барьер.** Действие: поставить на гекс временную стену, что блокирует
  прямые выстрелы (но не навес — см. выше). Чистый action (гекс + длительность), `Barrier`-сущность.
- **💡 Отражатель/дефлектор.** Щит, что **отбивает снаряд обратно** по физике. Зрелищно и
  читаемо (снаряд меняет владельца/направление). Контрит спам прямых выстрелов.

### Ось «структурный урон» (эксплуатирует модульность — то, чего нет у других)
- **💡 Кислота/коррозия — ослабляет СУСТАВЫ между кусками.** Не HP, а прочность Rapier-joints.
  Танк под кислотой **разваливается от собственного движения/тряски**. Уникально для модульных
  тел. `Corroded { jointMul, untilTick }`. Комбо: кислота → таран/грави (рассыпается от толчка).
- **💡 Бола/опутывание (entangle).** Joint, что **сцепляет гусеницы/куски** — физический «рут»:
  враг не может ехать, пока не порвёт. `Entangled`. Отличается от ЭМИ тем, что это физика, а не
  флаг (можно вырваться рывком/массой). Комбо с тараном/добиванием.

### Ось «мобильность» (репозиционирование как отдельное решение, не оружие)
- **💡 Рывок/буст.** Действие: короткий импульс ускорения. Сам по себе — манёвр уклонения;
  с тараном — разгон для удара. Даёт агенту ось «двигаться агрессивно vs аккуратно».
- **💡 Крюк-подтяжка к рельефу.** Трос к стене/тяжёлому обломку → подтянуть **себя** (не врага).
  Резкая смена позиции, объезд, затаскивание за укрытие. `GrappleAnchor { hexOrEid }`.

### Ось «таймер на поле» (отложенные решения, контроль пространства)
- **💡 Липучая мина/латч-бомба.** Цепляется к врагу (или гексу), взрыв с задержкой. Враг
  должен **среагировать** (стряхнуть рывком / уехать). Контрит каннибала/хиллера, что стоят на
  месте. `StickyBomb { detonateTick }`. Переиспользует `Explodable` из CLAUDE.md.

## ⚠️ Системная проблема: «сцепка» / тупик равновесия (важно для RL)

**Симптом:** два танка сошлись в упор, оба стреляют, снаряды **взаимно уничтожаются** в полёте.
Никто не получает урон, но и **выйти нельзя** — отвернёшься/поедешь → подставишься и тебя
добьют. Танки застревают в состоянии, из которого нельзя применить другую стратегию.

**Почему это плохо именно для RL:** тупик — это **равновесие Нэша**. Если стоять безопаснее,
чем уходить, политика PPO **выучит стоять** и застрянет в локальном оптимуме. Чтобы агент
научился размену/манёвру, нужно одновременно (а) дать **безопасный путь выхода** и (б) сделать
**стояние дороже**. Только рывка мало — он лечит (а), но без (б) агент всё равно предпочтёт тупик.

### (а) Пути выхода (disengage) — действия
- **✅ Рывок/буст** — резкий импульс, чтобы разорвать дистанцию/сорвать прицел. Ключевой нюанс:
  рывок **вбок ломает симметрию** — снаряды больше не идут лоб-в-лоб, перестают гасить друг
  друга → у одного проходит. То есть рывок не просто «убежать», а **сменить геометрию боя**.
  Возможно короткое окно сниженной уязвимости (i-frames) или просто скорость > точности врага.
- **✅ Отдача на выстрел** (из `mechanics.md`) — выстрел толкает назад. Бесплатное смещение:
  можно «выстрелить из сцепки» и отлететь. Дёшево, физично, читаемо.
- **Дым на себя** (🤔) — поставил дым, разорвал линию видимости, ушёл. Лечит (а), но это
  частичная наблюдаемость (см. ограничение RL выше).
- **Крюк-подтяжка** (💡) — затащить себя за угол/укрытие одним рывком.

### (б) Сделать стояние дороже (чтобы выход стал выгоден) — давление
- **💡 Перегрев/кулдаун пушки.** Нельзя стрелять бесконечно — после серии выстрелов **пауза**.
  В этот момент взаимное гашение снарядов прекращается → окно, когда выйти безопаснее, чем стоять.
  Создаёт ритм боя сам по себе. `Overheated { untilTick }`. Очень читаемо агентом.
- **💡 Конечный боезапас/энергия.** Патроны кончаются → вынужденный отход/перезарядка. Тоже
  ломает «стоять и стрелять вечно».
- **💡 Глобальное давление: сужающаяся зона / дрейф цели.** Battle-royale-кольцо или
  перемещающаяся точка интереса. Стоять на месте = штраф/смерть от зоны → тупик невыгоден
  на уровне reward, без правки боёвки. Самый сильный (б)-рычаг для обучения.
- **💡 Снаряды НЕ гасят друг друга полностью.** Пересмотреть саму причину: пусть пули
  пролетают сквозь друг друга (гасятся только крупные/ракеты), или гашение вероятностное.
  Тогда стояние в упор = взаимный размен уроном, а не безопасный тупик.

> Рекомендация: **рывок (а) + отдача (а) + перегрев или зона (б)**. Тогда у агента есть и
> инструмент выхода, и причина им пользоваться — PPO выучит размен/манёвр, а не зависание.

## Карта сочетаний (комбо — где появляется глубина)

Глубина не в отдельных пушках, а в том, что одно оружие создаёт **состояние**, которое
другое использует. Каждое состояние = компонент, каждая реакция = система (паттерн
`Explodable + Destroy` из CLAUDE.md).

| Создаёт состояние | Использует |
|---|---|
| Морозилка → `Chilled` | Грави/таран легко добивает медленного; держит на месте под DoT |
| Огнемёт → `Burning` | Детонация обломков-«бочек»; масло + огонь = горящая лужа |
| Обломки на поле | Магнит-подбор (каннибализм / хиллер-материал); грави-волна швыряет ими |
| Масло → `Slippery` гекс | Грави/таран спихивает дальше, враг не тормозит |
| ЭМИ → `Disabled(move)` | Грави/таран/добивание, пока стоит |
| Грави → сгрести обломки | Укрытие из мусора блокирует следующие выстрелы |
| Обломки на гексе | Грави-пушка кидает ими; детонация; блок прохода |
| Дым на гексах | Прикрыть отступление / рывок / спрятать мину |
